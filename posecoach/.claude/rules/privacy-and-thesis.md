# Privacy & Thesis Rules

## Video / Frame Privacy (GDPR)
- **NEVER log raw video frames** to disk, database, or any logging system.
- **NEVER store user webcam images** — only store derived pose keypoints (anonymized).
- Frame buffers are in-memory only and discarded after inference.
- If a user uploads a video for batch analysis, delete it after processing.

## Authentication Rules
- **JWT in httpOnly cookies ONLY** — NEVER store tokens in `localStorage` or `sessionStorage`.
- JWT algorithm: HS256.
- Cookie flags: `httpOnly=True`, `secure=True` (prod), `samesite="lax"`.
- Refresh tokens stored in DB with expiry — invalidate on logout.
- No sensitive user data in JWT payload beyond `user_id` and `exp`.

## Thesis Integrity
- Every implemented feature must map to at least one thesis evaluation metric (latency, accuracy, user study score).
- Do not implement features that cannot be measured or written about.
- All eval scripts live in `scripts/eval_*.py` — never inline evaluation logic in app code.
- Thesis tables are auto-generated by `scripts/export_thesis_tables.py` — do not hand-edit output CSVs.

## Data Sources
- Training data: Kaggle `hasyimabdillah/workoutfitnessvideo` — licensed for research use.
- RAG knowledge base: Exercise science papers — cite in thesis appendix.
- User study participants: anonymized IDs only (no names, emails in DB).

## API Keys
- Gemini key: environment variable `GEMINI_API_KEY` — never hardcode.
- OpenRouter (Qwen 3.6) key: environment variable `OPENROUTER_API_KEY` — never hardcode.
- Keys live in `.env` (gitignored) — never commit to git.
