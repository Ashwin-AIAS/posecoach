## Metric 5: SUS Score (≥ 70)
**SUS = System Usability Scale** (standard 10-question usability questionnaire, not a simple Likert):
- 10 alternating positive/negative statements, each rated 1–5
- Formula: `SUS = (sum_of_contributions) × 2.5` → score 0–100
- Target: ≥ 70 ("Good" usability)
- Minimum: **n ≥ 10 participants** (thesis requirement)
- Participant files: `data/eval/sus_responses/participant_XX.json`

## If a Metric Fails
| Failure | Action |
|---------|--------|
| mAP < 0.75 | P01: more training epochs or upgrade to yolo26s-pose |
| Latency > 100ms | Check `model.fuse()` before export; try imgsz=416 |
| Variance > 5% | Check EMA smoother is active; debug `score_smoother.py` |
| Chatbot < 80% | Expand knowledge base; tune RAG retrieval top-k |
| SUS < 70 | Identify lowest-rated statements; fix that feature |

## AIGym Baseline Comparison (Thesis Contribution)
Ultralytics `solutions.AIGym` is the off-the-shelf Ultralytics workout solution — track it as a baseline:

| Dimension | Ultralytics AIGym | PoseCoach |
|-----------|-------------------|-----------|
| Joints tracked | 1 angle triplet | All relevant joints per exercise |
| Calibration source | Hard thresholds | Fit3D mocap golden templates |
| Output | Rep count | 0–100 form score + coaching cues |
| Scoring depth | Binary (in/out) | Continuous (percentage of joints in range) |

Run AIGym on the same eval frames as `eval_form_consistency.py` to produce the comparison:
```python
from ultralytics import solutions
gym = solutions.AIGym(kpts=[5, 7, 9])  # shoulder, elbow, wrist for curl
# Compare: AIGym angle flag vs. our form_scorer output
```

**Thesis chapter line:** "Table X compares our multi-joint Fit3D-calibrated scorer against the Ultralytics AIGym single-angle baseline across 7 exercises."
This positions our approach as a deliberate upgrade, not an ad-hoc custom build.
